HumanAuth Vendor Integration Guide

Integrating HumanAuth into your application is straightforward. This guide demonstrates how easy it is for vendors to add HumanAuth to their existing authentication systems.

← Back to Architecture Overview

Step-by-Step Integration

1

Set Up API Access

Request an API key from the HumanAuth team to access the authentication service.

Store your API credentials securely in your application configuration.

2

Initialize Camera Access

Set up camera access in your application to capture video frames.

Use your preferred method to access the device camera (e.g., WebRTC in web applications).

3

Create Authentication Session

Start a new authentication session with the HumanAuth API.

Make a POST request to the /sessions endpoint with your API key in the headers.

4

Process Video Frames

Capture and send video frames to the HumanAuth API for processing.

Convert frames to base64 format and send them to the /sessions/{session_id}/process endpoint.

5

Display Challenges & Feedback

Show the user what they need to do to complete the authentication.

Parse the challenge information from the API response and display appropriate instructions to the user.

6

Handle Authentication Result

Process the final authentication result in your application.

When the API indicates successful authentication, grant access to the protected resource or continue with your application flow.

API Reference

POST /sessions

Purpose: Create a new authentication session

Headers: X-API-Key (your API key)

Response: Session ID for subsequent requests

POST /sessions/{session_id}/process

Purpose: Process a video frame and update authentication state

Headers: X-API-Key (your API key)

Body: JSON with base64-encoded frame

Response: Authentication status, challenge information, and progress

POST /sessions/{session_id}/reset

Purpose: Reset the current authentication session

Headers: X-API-Key (your API key)

Response: Confirmation of reset

Benefits of Integration

  • Enhanced Security: Protect your application from bots and automated attacks
  • Improved User Experience: No frustrating CAPTCHAs or puzzles
  • Flexible Implementation: Integrate with any application or authentication flow
  • Privacy-Focused: No personal data storage or identification
  • Cross-Platform: Works on web, mobile, and desktop applications

Security Considerations

When integrating HumanAuth with your systems, several security measures ensure safe and reliable operation:

  • CORS Protection: Configurable origin restrictions prevent unauthorized domains from accessing the API
  • API Key Authentication: Unique API keys for each vendor ensure only authorized systems can access the service
  • Rate Limiting: Prevents abuse through configurable request limits per vendor
  • Session Management: Unique session IDs prevent cross-session data access
