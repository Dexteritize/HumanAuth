The frontend captures camera frames and sends them to the backend via the API. The backend processes these frames using MediaPipe to detect faces and hands, verifying human presence through interactive challenges.

Core authentication logic using MediaPipe for face and hand landmark detection. 


# Key components in human_auth.py
- Face landmark detection
- Hand gesture recognition
- Challenge generation
- Anti-spoofing: micro-movements, blink patterns, 3D checks
Face Landmark Detection

Tracks 468 3D points to analyze expressions, head pose, and micro-movements.

Hand Landmark Detection

Tracks 21 points per hand for gesture recognition and challenge verification.

API Communication

REST API Endpoints

GET /api/config

Returns API configuration

POST /api/sessions

Creates a new authentication session

POST /api/sessions/{session_id}/process

Processes a frame and returns results

POST /api/sessions/{session_id}/reset

Resets the current session

POST /api/verify

One-time frame verification

The client initializes a session, captures camera frames, and sends them to the backend. The backend analyzes these frames using MediaPipe, issues challenges, and verifies responses. After completing all challenges, the system confirms human presence and returns an authentication result.

Security Considerations

Anti-Spoofing Defense Layers

Physical Presence
  • Micro-movement detection
  • Blink pattern analysis
  • Texture analysis
3D Spatial Analysis
  • 3D consistency checks
  • Depth verification
  • Head pose estimation
Interactive Challenges
  • Random challenge sequences
  • Unpredictable timing
  • Multi-modal verification
API Security
  • API key authentication
  • Rate limiting
  • Session validation

Attack Vector Defense

Attack Defense Effectiveness
Static Photo Movement detection, challenges High
Recorded Video Interactive challenges, 3D checks High
Deepfake/GAN Multi-modal verification Medium-High
3D Mask Texture analysis, hand verification Medium-High
API Abuse Rate limiting, authentication High
