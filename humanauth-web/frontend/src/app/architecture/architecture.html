HumanAuth System Architecture

HumanAuth is a comprehensive human authentication system that uses face and hand recognition to verify that a user is a real human rather than a bot or spoofing attempt. This page explains the architecture of the system, from the internal backend components to the API and frontend.

The HumanAuth system consists of three main components: a frontend Angular application, a REST API, and a backend Python server. The frontend captures camera frames and sends them to the backend via the API. The backend processes these frames using MediaPipe to detect faces and hands, and verifies that the user is a real human by issuing challenges and analyzing responses.

Backend Architecture

Flask Server (app.py)

The backend is built with Flask, a lightweight Python web framework. It provides REST API endpoints for the frontend to communicate with, as well as WebSocket support for real-time communication. 


# Key components in app.py
- REST API endpoints for session management and frame processing
- WebSocket handlers for real-time communication
- Authentication middleware for API key validation
- Error handling and response formatting

Human Authentication Engine (human_auth.py)

The core of the backend is the HumanAuth class, which handles the authentication logic. It uses MediaPipe to detect face and hand landmarks, and implements various anti-spoofing techniques. 


# Key components in human_auth.py
- Face landmark detection using MediaPipe
- Hand gesture recognition
- Challenge generation and verification
- Anti-spoofing techniques:
  - Micro-movement detection
  - Blink pattern analysis
  - 3D consistency checks
  - Texture analysis

MediaPipe Integration

HumanAuth uses MediaPipe, a cross-platform framework for building multimodal machine learning pipelines. It leverages MediaPipe's face and hand landmark detection capabilities to track 468 distinct points on the human face and 21 points on each hand in real-time.

Face Landmark Detection

Tracks 468 3D points on the face to analyze facial expressions, head pose, and natural micro-movements.

Hand Landmark Detection

Tracks 21 points on each hand to recognize gestures and verify interactive challenges.

Frontend Architecture

Angular Application

The frontend is built with Angular, a platform and framework for building single-page client applications. It provides a responsive user interface for the authentication process.

// Key methods in camera.service.ts

Handles communication with the backend API, including session management and frame processing. 


// Key methods in auth.service.ts

API Communication

REST API Endpoints

The backend exposes several REST API endpoints for the frontend to communicate with:

GET /api/config

Returns API configuration, including the API key and URL.

POST /api/sessions

Creates a new authentication session and returns a session ID.

POST /api/sessions/{session_id}/process

Processes a frame in the current authentication session and returns the result.

POST /api/sessions/{session_id}/reset

Resets the current authentication session.

POST /api/verify

Verifies a single frame without maintaining a session.

WebSocket Communication

In addition to REST API endpoints, the backend also supports WebSocket communication for real-time interaction during the authentication process. 


# WebSocket events
- connect: Client connects to the server
- disconnect: Client disconnects from the server
- start_auth: Client starts a new authentication session
- reset_auth: Client resets the current authentication session
- process_frame: Client sends a frame for processing

The authentication process begins when the user clicks "Start" on the frontend. The backend creates a new session and issues a series of challenges, such as turning the head or showing a specific hand gesture. The frontend captures frames from the user's camera and sends them to the backend for processing. The backend analyzes these frames using MediaPipe to detect face and hand landmarks, and verifies that the user is responding correctly to the challenges. After the user successfully completes all challenges, the backend confirms that the user is a real human.

Security Considerations

Anti-Spoofing Techniques

HumanAuth implements several anti-spoofing techniques to prevent attackers from using photos, videos, or masks to bypass the authentication:

Micro-Movement Detection

Analyzes natural micro-movements in the face that are difficult to simulate in static images or pre-recorded videos.

Blink Pattern Analysis

Detects natural blinking patterns that are hard to fake in static images or masks.

3D Consistency Checks

Verifies that the face landmarks are consistent with a 3D human face rather than a 2D image.

Interactive Challenges

Requires the user to perform specific actions that are difficult to predict and simulate.

Privacy Considerations

HumanAuth is designed with privacy in mind:

  • All processing happens locally on the server
  • No biometric data is stored long-term
  • Session data is temporary and deleted after authentication
  • No personal identification is required
