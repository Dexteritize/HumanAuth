HumanAuth System Architecture

HumanAuth is a comprehensive human authentication system that uses face and hand recognition to verify that a user is a real human rather than a bot or spoofing attempt. This page explains the architecture of the system, from the internal backend components to the API and frontend.

Vendor Authentication Integration

External applications can integrate with HumanAuth via:

  • REST API with API key authentication
  • Cross-Origin Resource Sharing (CORS) support

The HumanAuth system consists of multiple components working together to provide human authentication. The frontend captures camera frames and sends them to the backend via the API. The backend processes these frames using MediaPipe to detect faces and hands, and verifies that the user is a real human by issuing challenges and analyzing responses.

The system is designed to be flexible, allowing both the built-in demo page and external vendor applications to integrate with the authentication engine. The demo page provides a complete reference implementation, while the API endpoints enable custom integrations for various use cases.

Backend Architecture

Flask Server (app.py)

The backend is built with Flask, a lightweight Python web framework. It provides REST API endpoints for the frontend to communicate with. 


# Key components in app.py
- REST API endpoints for session management and frame processing
- Authentication middleware for API key validation
- Error handling and response formatting

Human Authentication Engine (human_auth.py)

The core of the backend is the HumanAuth class, which handles the authentication logic. It uses MediaPipe to detect face and hand landmarks, and implements various anti-spoofing techniques. 


# Key components in human_auth.py
- Face landmark detection using MediaPipe
- Hand gesture recognition
- Challenge generation and verification
- Anti-spoofing techniques:
  - Micro-movement detection
  - Blink pattern analysis
  - 3D consistency checks
  - Texture analysis

MediaPipe Integration

HumanAuth uses MediaPipe, a cross-platform framework for building multimodal machine learning pipelines. It leverages MediaPipe's face and hand landmark detection capabilities to track 468 distinct points on the human face and 21 points on each hand in real-time.

Face Landmark Detection

Tracks 468 3D points on the face to analyze facial expressions, head pose, and natural micro-movements.

Hand Landmark Detection

Tracks 21 points on each hand to recognize gestures and verify interactive challenges.

API Communication

REST API Endpoints

The backend exposes several REST API endpoints for the frontend to communicate with:

GET /api/config

Returns API configuration, including the API key and URL.

POST /api/sessions

Creates a new authentication session and returns a session ID.

POST /api/sessions/{session_id}/process

Processes a frame in the current authentication session and returns the result.

POST /api/sessions/{session_id}/reset

Resets the current authentication session.

POST /api/verify

Verifies a single frame without maintaining a session.

Backend Authentication Process

Session Management

Creates and tracks authentication sessions

  • Generates unique session IDs
  • Initializes HumanAuth instance for each session
  • Maintains session state
→
Frame Processing

Processes incoming camera frames

  • Decodes base64 image data
  • Resizes frames for optimal processing
  • Passes frames to HumanAuth engine
→
Face & Hand Detection

Uses MediaPipe to detect landmarks

  • Detects 468 face landmarks
  • Detects 21 hand landmarks
  • Analyzes landmark positions and movements
→
Challenge Management

Issues and verifies challenges

  • Generates random challenges (turn head, show hand)
  • Verifies user responses to challenges
  • Tracks challenge completion progress
→
Authentication Verification

Determines if user is human

  • Applies anti-spoofing techniques
  • Calculates confidence score
  • Returns authentication result

The authentication process involves coordination between the client application (either the demo page or a vendor integration) and the backend server. The client initializes a session, captures camera frames, and sends them to the backend for processing. The backend analyzes these frames using MediaPipe to detect face and hand landmarks, issues challenges, and verifies that the user is responding correctly. After the user successfully completes all challenges, the backend confirms that the user is a real human and returns an authentication result.

The demo page provides a complete reference implementation with built-in UI components for displaying challenges and visual feedback. Vendor applications can integrate with the same backend API but implement their own UI and business logic to fit their specific needs. Both integration paths follow the same core authentication flow but may differ in how they present challenges and handle the authentication result.

Security Considerations

HumanAuth is designed with security and privacy as core principles. The system implements multiple layers of protection against spoofing attempts while ensuring user privacy through careful data handling practices.

Anti-Spoofing Defense Layers

Physical Presence Verification
  • Micro-movement detection
  • Natural blink pattern analysis
  • Texture analysis to detect masks/photos
3D Spatial Analysis
  • 3D consistency checks
  • Depth ratio verification
  • Head pose estimation
Interactive Challenge Response
  • Randomized challenge sequences
  • Unpredictable timing requirements
  • Multi-modal verification (face + hands)
API Security
  • API key authentication
  • Rate limiting
  • Session-based authentication

Attack Vector Defense Matrix

Attack Vector Defense Mechanism Effectiveness
Static Photo Micro-movement detection, blink analysis, challenge response High
Pre-recorded Video Interactive challenges, timing analysis, 3D consistency High
Deepfake/GAN Multi-modal verification, texture analysis, challenge complexity Medium-High
3D Mask Texture analysis, micro-expression detection, hand verification Medium-High
API Abuse Rate limiting, API key authentication, session validation High

Privacy Protections

Temporary Data Processing

All biometric data is processed in real-time and not stored long-term

Server-Side Processing

All sensitive processing happens on the server, not in the browser

Anonymous Authentication

No personal identification or account linking required

Session Isolation

Each authentication session is isolated and data is deleted after completion

Vendor Integration Security

When integrating HumanAuth with vendor systems, several security measures ensure safe and reliable operation:

  • CORS Protection: Configurable origin restrictions prevent unauthorized domains from accessing the API
  • API Key Authentication: Unique API keys for each vendor ensure only authorized systems can access the service
  • Rate Limiting: Prevents abuse through configurable request limits per vendor
  • Session Management: Unique session IDs prevent cross-session data access
