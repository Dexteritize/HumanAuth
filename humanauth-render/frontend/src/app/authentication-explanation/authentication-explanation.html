How HumanAuth Authentication Works

HumanAuth uses face and hand recognition to verify humans vs. bots.

Multi-Modal Authentication System

HumanAuth implements a sophisticated liveness detection system using multiple detection methods to ensure that the user is a real human and not a bot or spoofed image.

🔍

3D Consistency Checking

  • Tracks face mesh landmarks through head rotation
  • Calculates depth ratios between facial features
  • Verifies that 3D relationships remain consistent (real faces vs. photos/screens)
⏱️

Temporal Analysis

  • Detects micro-movements (real faces never stay perfectly still)
  • Analyzes natural breathing patterns visible in slight head movements
  • Monitors eye blink frequency and pattern (humans blink 15-20 times/min in irregular patterns)
🎯

Active Challenges

  • Issues random prompts: blink once, show peace sign, show five fingers
  • Measures response time (humans ~200-300ms, video playback has delays/unnatural timing)
  • Requires completion of multiple challenges for authentication
🔬

Texture/Frequency Analysis

  • Analyzes pixel patterns at multiple scales
  • Detects specific frequency characteristics of real skin vs. printed paper or screens
  • Identifies Moiré patterns from screens

Challenge Types

👁️

Blink Once

The system detects when the user blinks by measuring the eye aspect ratio (EAR).

✌️

Show Peace Sign

The system detects when the user shows a peace sign by analyzing hand landmarks.

🖐️

Show Five Fingers

The system detects when the user shows all five fingers by analyzing hand landmarks.

Security Considerations

🛡️

Anti-Spoofing

The system uses multiple methods to detect spoofing attempts, such as photos, videos, or masks.

🔒

Privacy

All processing is done locally or on secure servers, and no personal data is stored long-term.

⚡

Performance

The system is optimized for low latency and can run on standard hardware.
